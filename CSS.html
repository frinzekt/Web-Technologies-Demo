Cascading Style Sheets (CSS)

Brief History & Features

Taken from CSS History, CSS1

1991 Basic Creation (HTML 1.0)

With the creation of HTML, there was a need to add presentation to the content, so HTML was extended with tags that adds presentational value

Some developers created browsers that incorporated stylesheets in the browser, but did not get close to what CSS is today, although this provided the inspiration for potential CSS specifications

Late 1994 - Early 1995: CSS Introduction

CSS was proposed and was called Cascading Stylesheets that introduced basic,but powerful Features:

  • Class and ID Selectors, and Declarations
  • Font, Letter Spacing and Alignment
  • Colors
  • Tables and Frames
  • Spacing and The Box Model (explained here)

1998: CSS2

CSS was improved to add more features:

  • positioning (absolute,relative,fixed)
  • Z-index (Defines which element is on top of another when overlapping)
  • Concept of Media Type (Accounting for different screen)
  • Bidirectional text
  • New Font Properties

1999-2018:CSS3

CSS3 introduced modules that are divided into multiple stage of development as mentioned here and here CSS3 adds a lot more features that helps in aligning, and creating "modern" designs that are used a lot in these days such as:

  • CSS3 Attribute Selectors

    This allows dynamic designs based on certain attributes of an element such as. 

    E[attr^=val]
										eg.          a[href^='http://sales.']{color: teal;}

    This uses the wildcard "^" to indicate starting. There are other wildcards such as "$" for end, "*" for anywhere match

  • Pseudo-classes

    Allows some dynamic design that corresponds to certain JS events without the need to use JS such as :link, :visited, :hover, :active, :focus

  • CSS 3 Colors

    Prior to CSS3, it can only support hexadecimal format. Now there are various ways to describe color such as RGB, HSL, HSLA, and Opacity

    • There are many more features implemented which can be found here in the 50 new CSS tricks, but these are the most important.

  • Separation of Content and Design / Separation of Work

It also divides the work for web developers and web designers. Hence both teams can work independently, but still achieving the same goal - developing/designing a good-looking and functional website.

Box Model

The box model is one of the most important to understand in order to make sure elements are in their corresponding right sizes. The box model displays the sizes of individual element parts that affects its sizes. This can be seen in the adjacent images taken from here

It is composed of :

  • Margin

    It defines the spacing of the element from the screen or from other elements (depending on the positioning css attribute)

  • Outline (related, but does not belong to the box model)

    Just like the border, excepts adjusting the size of the outline makes it extends inwards which means that the total size of the element does not actually increase unlike border. Due to irrelavance in size, it does not belong to the box model.

  • Border

    It defines the size of line that surrounds the whole element.

  • Padding

    It defines the size of the outer shell of the content

  • Content Dimension

    It defines the actual size of the content, and can be adjusted using the properties width,and height

CSS Selectors

CSS Selectors is the most important features in CSS as it is the main method to link design to HTML elements. CSS Selectors are classified in the following:

  • Tag Selectors

    Applicable for directly applying designs directly to an HTML tag

  • Class Selectors

    Applicable for directly applying designs toward a group of elements.

  • ID Selectors

    Applicable for directly applying designs toward as specific element

  • Pseudo Class Selectors

    Applicable for applying design for particular scenario such as :first-child, :last-child

  • Attribute Selectors

    Applicable for applying designs for elements that has a specific attribute. Very useful for applying design such as all checkbox that has been ticked
