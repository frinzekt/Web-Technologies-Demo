What is it?

HTML is a standard markup language for documents to be displayed in web browser. It serves as the layout of the document's content in a web browser by using what's called Document Object Model (DOM) Tree

History

1991-1993: Basic Creation (HTML 1.0)

Tim Berners Lee created the HTML in the late 1991, and released HTML 1.0 in 1993. This is created to share information that is readable and accessible via web browsers

1994-1995: Official Release of HTML (HTML 2.0)

Official publication of HTML as HTML 2.0 which contains all the features of HTML 1.0 along with additional features. HTML became the official standard markup language for designing and creating websites, and refined various core features of HTML which made

1995-1997: CSS Introduction and LOTS OF CONFLICTED GOALS (HTML 3.0)

HTML 3.0 publication is a huge conflict. Many parties wanted different features, and implemented those features which became a problem as there is no single standard. A HTML document could look different in different browsers. Even though HTML 3.0 was rejected multiple times, there was an agreed upon list of implemented features at the end of the 1997:

  • Content NOT Presentation Markup

    Establishment of HTML as a structural/content markup rather than to make "things pretty". This is supported by the need of platform independence (support for different screen sizes, devices, presentation formats). This was particularly difficult as presentation is most of the time linked to expression of intention/message.

  • Support for Cascading Style Sheets The key takeaway here is that there needs to be a way to express design without incorporating with structure.

  • Support for Non-Visual Media

    Inline figures are polished especially with the use of table markups

  • Support for Different Ways Of Creating HTML

    There was an introduction of creating HTML with a variety of tools such as word processing format or other SGML application. This is interesting as it is possible to design a basic website in word processor such as Microsoft Word as an export. What You See Is What You Get (WYSIWYG) feature is an important piece for reaching more user of HTML as it is very straightforward

1988-1999: More removal of Presentational Tags, Object Multimedia, and Smart Forms(HTML 4.01)

Basic CSS has been introduced in order to separate some of the presentational layer from HTML as well as trimming some of the old tags such as: <ISINDEX> <APPLET> <CENTER> <FONT> <BASEFONT> <S>. Most of these are replaced with their css counterparts, as the tag name suggest, these are old html tags that gives presentation. Also, multimedia object are replaced to be more general with <OBJECT>

Moreover, there some new tags that were added such as : <Q> <ACRONYM> <INS> <DEL> <COLGROUP> <FIELDSET> <LABEL> <BUTTON> . Most of these complements the existing tags specifically the "phrasal" (blockquote, strong, code, em) s and the form/input tags.

2012 - Today: (HTML 5.0)

One of the biggest change for the web technology as a whole. HTML5 is the integration of several different technology (CSS, JavaScript, and server-based technologies) so this update made it easier to update or integrate with the other technologies

  • Semantic markup: New tags that describe parts of a document such as<NAV>, <ARTICLE>, <SECTION>, <HEADER>, and <FOOTER>.
  • New form elements: HTML5 forms have some major updates. There are several new versions of the <INPUT> element, allowing users to pick colors, numbers, e-mail addresses, and dates with easy-to-use elements.
  • Media elements: HTML5 has native support for audio and video with tags similar to the <IMG> tag.
  • canvas tag:This allows creation of object and interface elements that are useful for gaming, charts and visualizations.

Most Important Tags Of HTML

These are the most important features of HTML that will be seen in most websites to follow

  • Meta Tags

    These tags are very useful for SEO (search engine optimisation) as it will help webcrawlers in finding the site content. There is only two meta tags used in this website to specify the content size (to properly shift to mobile) and character content.

  • Division Tags

    These tags are probably the most important tag as it determines the main structure of the DOM. This has many names such as <DIV> <NAV>, <ARTICLE>, <SECTION>, <HEADER>, and <FOOTER>, but their difference of use is only for semantic.

  • Anchor Tags

    This tags facilitates linkage towards other websites or sections. Take the navbar for example, and this that takes you to the HTML history

  • Form Tags

    Most websites will have some sort of input system such as a contact form, search, or login which makes these types of tags essential. Check the JS Demo

    There are a lot of form tags a few of them are <input>, <textarea>, <button>, <select>, <option>, <optgroup>, <fieldset>, <label>, <output>

  • Script and Stylesheet Linking Tags

    These tags are used in order to integrate CSS and JavaScript to HTML. such as
